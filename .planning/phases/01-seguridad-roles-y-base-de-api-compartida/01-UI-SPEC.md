# Phase 1 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

**Scope note:** This phase's frontend surface is almost entirely non-visual (route guard, HTTP client, role plumbing). The only *new* rendered UI is a single placeholder view (`/proximamente`) shown to `cliente`/`ti` roles per D-06, plus a behavior contract for `<RequireRole>`'s redirect. This spec covers those two things only. It does **not** restyle Login, AppLayout, AdminLayout, or any existing operator/admin view — those remain untouched (out of scope per 01-CONTEXT.md).

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none — no `components.json`, no Tailwind/PostCSS config detected in `agrodroid/web`. The app uses hand-authored CSS with CSS custom properties (`agrodroid/web/src/styles/Usuario/theme.css`), one `.css` file per page/component. |
| Preset | not applicable |
| Component library | none (plain React + CSS) |
| Icon library | none currently in `package.json` — do not introduce one for this phase; the placeholder view needs no icon |
| Font | **Inter** for the new placeholder view only, per the locked `DESIGN.md` token contract (`typography.heading-4`, `typography.body-md`, `typography.caption`). Fallback stack: `Inter, -apple-system, BlinkMacSystemFont, "Segoe UI", sans-serif`. **Known tension, documented not silently resolved:** the rest of the app (`theme.css` → `--font-body: Arial, Helvetica, sans-serif`) is not Inter. Harmonizing the whole app's typeface is out of scope for this phase (course deadline, minimal placeholder-only phase per CONTEXT.md). Do not retrofit existing pages to Inter in this phase — that decision is deferred. |

**Why `DESIGN.md` tokens apply narrowly here:** `DESIGN.md` is locked project-wide as the token source of truth (per project setup), but this phase creates exactly one new visual surface — the placeholder view. That view adopts `DESIGN.md`'s literal tokens (spacing, radius, typography, card and button component definitions) as directed. Existing pages keep their current `theme.css` variables (`--accent-vine` green identity, Arial) unchanged; reconciling the two systems app-wide is a future-phase decision, not this one.

| Token | Value | Usage |
|-------|-------|-------|
| xxs (`spacing.xxs`) | 4px | Gap between eyebrow badge and title |
| xs (`spacing.xs`) | 8px | Gap between title and body copy |
| sm (`spacing.sm`) | 12px | Internal button padding (vertical) |
| md (`spacing.md`) | 16px | Gap between body copy and the "Cerrar sesión" button |
| lg (`spacing.lg`) | 20px | Button internal padding (horizontal, per `button-secondary`) |
| xl (`spacing.xl`) | 24px | `card-base` internal padding |

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px (`typography.body-md`) | 400 | 1.5 |
| Label | 13px (`typography.caption`) | 400 | 1.4 |
| Heading | 22px (`typography.heading-4`) | 600 | 1.3 |
| Display | N/A — not used. This view has no display-scale text; the card title (Heading, 22px) is the largest text on screen. |

## Interaction & Guard Behavior

*(Added beyond the standard template sections — needed to keep `<RequireRole>`'s visible behavior consistent, per phase context.)*

- **No-token case (unauthenticated):** `<RequireRole>` redirects instantly to `/login` via `<Navigate replace />`. No spinner, no flash of protected content — the check is a synchronous `localStorage` read, so there is no network round-trip to wait on.
- **Wrong-role case (authenticated, cross-role URL access):** redirect instantly (same `<Navigate replace />` pattern) to the user's own role home (`admin` → `/admin_dashboard`, `monitor` → `/dashboard`, `cliente`/`ti` → `/proximamente`). Do not render the blocked route's content even momentarily. If a visible message is included, use the Error state copy above as a transient toast/alert immediately before the redirect fires — do not block the redirect waiting for dismissal (`alert()` is acceptable given the existing app's error-handling convention of using `alert()` for user-facing errors, but must not delay navigation).
- **Placeholder view itself:** renders immediately on route entry, no loading state — it has no data fetch, just static copy inside a `card-feature`-style container, centered in the `app-shell__content` area (reuse the existing `.app-shell` shell/layout classes from `theme.css`, do not introduce a new page shell).
- **Route path:** `/proximamente` (new, added to `App.tsx`'s route table alongside `/dashboard/*` and `/admin/*`).
